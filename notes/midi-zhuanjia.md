# 2026-07-23
<!-- DAILY_CHECKIN_2026-07-23_START -->

Moss 的核心架构遵循了典型的\*\*分层隔离与确定性验证设计\*\*。它的实现原理是：\*\*“将模糊的 AI 意图转为确定性的规范数据，利用链上沙盒演练校验，最后将控制权交还给用户。”

一、 核心技术流转架构

Moss 的内部运作是一个严格的 5 步流水线，将“自然语言输入”一步步安全地降维并转换为“区块链原始交易”：

AI Agent 意图 (自然语言)

↓

\[ 1. Discover \] 语义向量匹配/关键词索引 ➔ 锁定协议与函数

↓

\[ 2. Load \] 契约化模板转换 (Human/Agent Standard Format)

↓

\[ 3. Action \] 构建不可变的交易计划 (Plan)

↓

\[ 4. Simulate \] 链上节点 Fork & Trace 模拟 ➔ 输出 Effects & Warnings

↓

\[ 5. Wallet \] 交易计划与效果清单送入用户钱包签名 (No Private Keys inside Moss)

二、 底层三大实现机制

1\. 契约加载与数值安全规整

\* **十进制字符串防溢出：** LLM（大语言模型）非常擅长处理文本，但在做 $10^{18}$ 这种精确的大数/浮点数乘除时极易出错。Moss 在契约层（Load）强制规定：Agent 输入金额必须使用 `"1.5"` 这样的\*\*十进制字符串\*\*，而将精度转换（Decimals Alignment）、大数编码（BigNumber conversion）完全交给 Moss 底层确定性的代码去处理。

\* **符号策展（Curated Catalog）：** 链上 ERC-20 代币的 `symbol()` 可以伪造（例如同名假币）。Moss 内部维护了一个受信任的 Token 目录，阻止 AI 直接使用任意链上返回的符号进行交易映射。

2\. 链上沙盒仿真与 Trace 校验

Simulate 是 Moss 最硬核的模块。AI 生成交易 Plan 后，Moss 会对目标区块链节点发起一个\*\*模拟调用（如通过 `eth_calldebug_traceCall` 或本地 Fork 节点的 State Override）：

\* **提取状态变更 (Effects)：** 仿真器不会去听 AI 的汇报，而是直接抓取链上沙盒演练后的底册变更日志（State Diff），精准算出此交易会导致你的钱包：\*\*支付多少 Gas、扣除多少 Token A、实际收到多少 Token B\*\*。

\* **拦截异常告警 (Warnings)：** 在沙盒演练过程中，如果发生合约 Revert（交易失败）、授权额度异常（Allowance Exceeded）、滑点越界或检测到疑似蜜罐（Honeypot）逻辑，Moss 会强行触发 `Warning` 标签，阻止交易下一步进行。

3\. 交易计划 Hash 与状态对齐

\* **Plan 不可篡改：** 当 Moss 构建好一个写操作计划（Plan）并经过仿真后，会为这个 Plan 生成一个唯一哈希`planHash`）。

\* **对账一致性检查：** 只有当 `planHash` 校验通过、模拟结果完全无 `Warning`，且实际演练输出的 `effects`（资产变动）与 Agent 承诺给用户的意图完全吻合时，Moss 才会允许该 Plan 输送到前端界面。

三、 Moss 的安全防御逻辑

针对 AI Agent 在 Web3 中最常见的攻击路径，Moss 在原理层面设计了针对性的防线：

| 潜在威胁/攻击 | 传统 Agent 处理方式 | Moss 的实现原理与防御 |

| **提示词注入 (Prompt Injection)**<br>

<br>\*(黑客诱导 AI 给出“将资产转走”的指令)\* | AI 拥有私钥，直接执行并转走资产。 | **剥离私钥：** Moss 不持有私钥。即使 AI 被注入，Moss 构建出的 Plan 也会在沙盒模拟中暴露出 `Effects: -100 ETH` 的风险，并且需要\*\*人类私钥签名\*\*才能发出，攻击直接失效。 |

| **Calldata 编码机制幻化**<br>

<br>\*(AI 组装出的 hex calldata 差了一位)\* | 交易发送到链上失败，白白浪费 Gas 费甚至触发合约漏洞。 | **意图模板隔离：** AI 不接触底层的 ABI/Calldata 组装过程，AI 只填充高层参数，由 Moss 的确定性 SDK 完成底层逻辑打包。 |

| **链上 MEV / 抢跑 / 滑点地雷** | 链上深度不够导致高滑点，资产严重受损。 | **Simulate 拦截：** 在把交易送交钱包前，沙盒演练会精准捕获滑点损失，若超出阈值直接生成 `Warning`，拒绝进入签名阶段。 |

这份关于 Ethereum 开发环境 Hardhat 官方仓库的简要介绍

项目结构

Hardhat 采用的是典型的 Monorepo（多包存储库） 架构，使用 `pnpm` 进行工作区（Workspace）管理。

`packages/`核心代码库。Hardhat 的核心功能、官方插件（如 `hardhat-ethershardhat-viem` 等）都作为独立的包子目录存放在这里。

`docs/`存放项目官方文档的源文件。

`changeset/`用于自动化管理版本号和生成 ChangeLog 的配置目录。

README

一开篇就强调自己是“面向专业人士的以太坊开发环境”，由 Nomic Foundation 为以太坊社区打造。

摒弃了复杂的长篇大论，直接给出一行初始化命令`npx hardhat --init`，引导开发者通过交互式命令行快速上手。

Docs

文档直接内置在 `docs/` 目录中，意味着每次核心代码的重大变更都会与文档修改同步进行。

Issues & Pull Requests

Issues ：数量较多，通常包含新功能的提案、Bug 反馈以及关于 v3 新特性的讨论。

Pull Requests：代码合入非常频繁。项目采用了标准的开源协作流，通过 `CONTRIBUTING.md` 规范社区贡献，并使用 `.claude` 和 AI Agent（如 `AGENTS.mdCLAUDE.md` 中所示）来辅助维护者进行代码规范检查。

发现

Hardhat 正在将其底层的以太坊模拟器/运行时迁移或升级到基于 Rust 构建的高性能组件上，以极大提升编译和测试速度。

全栈 TypeScript：项目语言占比中 TypeScript 占了 95.9%。虽然它是用来开发 Solidity（2.0%）智能合约的工具，但它自身的工具链完全是高质量、强类型的 TS 编写。

仓库中出现了 `CLAUDE.mdAGENTS.md` 以及 `docs(hhu): sync the add-hhu-util skill...`。这表明 Nomic Foundation 正在积极拥抱 AI 编码助手，并为 AI Agent 提供了项目内专属的上下文指令文件。

# 2026-07-18
<!-- DAILY_CHECKIN_2026-07-18_START -->







简介:

Moss 是 Monad 生态打造的 AI Agent 交易能力框架与安全验证工具.

它扮演着 AI Agent 与区块链底层之间“安全网关”的角色。通过将复杂的链上交互、ABI 编码和风险模拟进行标准化封装，Moss 让 AI 既能理解并调用各种链上协议。

核心问题:

在 Moss 出现之前，让 AI Agent 代替人类进行 Web3 操作面临,

资金归零的“幻觉风险”： 区块链交易是不可逆的。AI 在理解复杂的智能合约、计算 Token 小数位（Decimals）或编码 Calldata 时，一旦发生微小的逻辑幻觉，就会导致交易失败甚至资金永久丢失。

私钥保管的“信任僵局”： 如果把钱包私钥直接托管给 AI，一旦 Agent 被黑客通过提示词注入（Prompt Injection）攻击，或者其自身代码出现漏洞，用户的全部资产将毫无防备地被洗劫。

协议对接的“开发黑洞”： Web3 协议千差万别，让 AI 每次都去生啃不同项目的 ABI 极其低效。缺乏一个统一的、能让 AI 秒懂的“协议操作说明书”。

核心能力:

Moss 通过一套“能力发现,意图加载,沙盒模拟,钱包隔离”克服了上述问题：

语义化能力检索： 允许 AI 通过自然语言（如 _“帮我把 10 个 Token A 换成 B”_）直接检索并匹配到对应的链上协议工具。

意图与参数标准化： 自动处理大数运算和精度对齐，把复杂的合约调用参数翻译成 AI 和人类都能一眼看懂的“意图模板”。

链上仿真与对账： 在交易真正提交前，在\*\*链上沙盒\*\*中进行仿真执行。Moss 会提前抓取潜在的滑点、归零风险（Warning），并直观展示这笔交易会导致你钱包里资产发生什么变化（Effects）。

核心安全原则——“只构建，不签名”： Moss 帮 AI 构建好完美的交易 Plan，但它\*\*没有私钥，无法签名\*\*。最终的放行权永远在人类（通过安全钱包）手中。

未来可能应用的场景:

自动化全网套利与策略执行,游戏 AI 自动化打金与资产管理,链上 DAO 智能财务与自动执行秘书.......

个人理解:

在我看来，Moss 的本质是为 Web3 时代的 AI 交互重构了“最小特权原则”。

这种“AI 动脑，Moss 铺路，人类把关的架构，是未来 AI Agent 能够真正大规模进入加密金融、管理真实资产的路径。真正变成了能够安全调用一切智能合约的链上“超级赛博代理”。
