# 二、理论知识学习

## 1\. 区块链钱包基本概念

在传统互联网中，用户身份通常由账号和密码进行管理，而在 Web3 世界中，用户身份由区块链钱包地址表示。

钱包本身并不存储数字资产，而是保存用户访问区块链账户所需要的密钥信息。

区块链钱包主要包含：

-   私钥（Private Key）
    
-   公钥（Public Key）
    
-   钱包地址（Address）
    
-   助记词（Mnemonic Phrase）
    

其中：

私钥是用户控制资产的唯一凭证，通过密码学算法生成公钥，再由公钥经过哈希计算得到钱包地址。

整个过程：

```
随机数
 ↓
私钥 Private Key
 ↓
公钥 Public Key
 ↓
钱包地址 Address
```

钱包地址类似于传统银行账户号码，而私钥则类似于银行卡密码，但区块链中的私钥权限更高，一旦泄露，资产将无法恢复。

-   可以调用智能合约。
    

例如：


该地址属于用户控制账户。

## 2\. 合约账户（Contract Account）

由智能合约代码控制。

特点：

-   没有私钥；
    
-   由代码逻辑管理；

-   是去中心化应用（DApp）的核心。
    

例如：

智能合约可以实现：

-   NFT交易；

# 五、链上交易流程学习

通过学习了解到，一笔区块链交易完整流程如下：

```
用户发起交易

↓

钱包生成交易信息

↓

私钥进行数字签名

↓

发送至区块链节点

↓

矿工/验证节点验证

↓

交易进入区块

↓

区块确认

↓

链上状态更新
```

-   计算资源消耗；

**三、核心概念理解（重点）**

**1\. 钱包的本质：不是“存钱工具”，而是“身份系统”**

在传统互联网中，用户身份依赖中心化账号体系（如手机号、邮箱、用户名）。

而在 Web3 中：

钱包 = 去中心化身份（Decentralized Identity）

钱包的核心作用不是“存储资产”，而是：

-   生成唯一身份（Public Address）
    
-   提供签名能力（Signature）
    
-   控制链上状态变更权限
    

换句话说：

区块链并不“认识人”，只认识“能签名的密钥”

* * *

**2\. 地址（Address）与私钥（Private Key）的关系**

当前理解如下：

-   私钥：唯一控制权，类似“终极密码”
    
-   公钥：由私钥推导生成
    
-   地址：公钥进一步哈希后的结果（用户可公开）
    

关键认知：

地址是公开身份，但控制权永远只存在于私钥

因此：

-   地址可以公开（用于收款 / 交互）
    
-   私钥必须绝对保密（资产安全核心）
    

* * *

**3\. 签名机制：Web3的信任基础**

Web3中不存在“账号密码登录”，取而代之的是：

Sign (签名) = 用私钥证明“我是这个地址的拥有者”

签名的作用：

-   证明身份
    
-   授权交易
    
-   防止篡改
    

可以理解为：

区块链世界的“身份验证 = 数学证明”，而不是中心化验证

* * *

**4\. 钱包在区块链系统中的位置**

当前形成的系统认知：

用户 → 钱包 → 签名 → 交易 → 区块链网络 → 状态更新

可以理解为：

-   钱包是用户与区块链之间的“唯一接口”
    
-   所有链上行为必须经过钱包签名授权
    
-   区块链本身不执行“用户意图”，只执行“已签名交易”
