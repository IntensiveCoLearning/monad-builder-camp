# Day2 打卡笔记：工具准备与 Builder 身份，开启 Web3 远程协作之路

## 今日学习目标复盘

今天围绕三大既定学习目标推进学习，整体收获清晰：

1.  梳理罗列好了 Web3 日常开发、远程协作必备的工具清单，规划好了后续逐个完成软件安装、注册适配账号的安排；
    
2.  细致区分了 DevRel、Builder、生态连接者三类岗位的定位差异，弄懂了各自日常工作侧重，明晰了 Builder 在生态建设当中的核心职责；
    
3.  了解了 Co-learning 协同学习模式，做好了规划，后续会把日常学习产出、实操过程留存下来，打造持续更新的 proof of work 工作证明。
    

## 具体学习内容收获

### 一、手册研读感悟

1.  阅读《Web3 实习手册 - 远程工作指南》之后，掌握了 Web3 行业远程协作特有的工作习惯、沟通节奏，跟着指南记下了各类工具的详细安装步骤，避开了不少初次配置容易踩到的坑；
    
2.  翻看岗位全景介绍板块，完整摸清了 Web3 领域细分岗位的分工情况，理清了 Builder 需要对接的上下游角色，明白了日常工作需要配合 DevRel 做生态推广、联动生态连接者拓展合作。
    

### 二、工具筹备规划

按照学习给出的清单整理了工具筹备任务，拆分了落地步骤：

1.  社交协作端：准备完成 X（Twitter）、Telegram、Discord 账号搭建，这三个平台主要用来跟进项目官方动态、加入开发者社群、日常对接海内外同行开展远程沟通；
    
2.  代码归档端：完善 GitHub 主页，后续将自己练习代码、项目迭代的仓库统一规整，让 GitHub 主页成为自己很重要的工作证明载体；
    
3.  文档沉淀端：选定 Notion 作为自己日常的学习记录本，用 Google Docs 协作完成团队共享文档任务，系统化留存每日学习笔记、任务台账；
    
4.  开发增效端：对比 Cursor、Claude Code 等多款 AI 编码工具的特性，挑选适配自身编码习惯的工具，借助 AI 优化代码编写、自查 bug、撰写文档的效率。

# Web3 学习打卡笔记

日期：2026.07.06

今日完成学习内容：通读《Web3 实习手册》开篇前言板块，摸清了这本手册的创作背景、初衷以及适配的学习人群，收获颇丰。

## 学习收获

### 1\. 明晰手册定位

这本手册面向想要入行 Web3 的实习生搭建了完善的学习体系，学习内容覆盖区块链基础、以太坊生态、行业细分岗位、实操技巧、面试备考多个模块，能够全方位帮助新人适配 Web3 领域，顺利完成入行过渡。

### 2\. 了解新人入行普遍痛点

我梳理出新手踏入 Web3 常见的四处难题：

①不清楚 Web3 基础知识的起步学习路径；

②晦涩的行业专有名词、技术概念理解门槛偏高；

③没有成型的职业规划，对未来发展方向感到迷茫；

④很难寻觅靠谱的学习素材以及落地实操的契机。

手册正是针对以上痛点，由 ETHPanda、LXDAO 社区联合编写，给到体系化的学习路线与职业指引，完美适配现阶段我的学习需求。

### 3\. 找准自身适配位置

手册的目标读者主要分为在校感兴趣大学生、刚踏入行业无从下手的 Web3 新人两类，我正好匹配受众范围，后续可以顺着手册左侧目录循序渐进学习：先完成入门导读部分筑牢底层基础，再拓展行业知识、阅读前辈访谈汲取经验，最后着手准备求职面试。

## 后续学习计划

1.  接下来优先攻克入门导读栏目，逐个吃透区块链基础概念、以太坊概览、行业赛道、日常工作模式这几小节，搭建初步的知识框架；
    
2.  做学习过程笔记专门整理难懂的专业术语，降低后续学习的理解难度；
    
3.  跟着手册指引搜集对应的实践资源，尽量尽早获得实操机会，避免只停留于纸面理论。
    

## 今日小感悟

Web3 领域发展速度很快，零散碎片化自学很容易走弯路，借助社区打磨完成的系统化手册开展学习，能够规避很多新手常见的踩坑环节。往后坚持每日打卡稳步推进，一步步搭建自己的行业知识储备，朝着适配 Web3 实习岗位的目标推进。
