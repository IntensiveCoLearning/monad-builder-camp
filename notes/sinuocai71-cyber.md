# 以太坊协议层研究计划 EPF 经验分享

## 核心观点

EPF 是以以太坊协议层的深度研究计划，核心在于理解并改进以太坊底层机制，而非应用开发

1.  成功参与的关键在于选定小范围持续深耕、提交易于 Review 的 PR，并展现对以太坊生态的热情与理解
    
2.  该领域学习周期长、职业路径非传统，更适合出于技术兴趣与开源贡献精神的参与者
    

## EPF 计划介绍

### 计划性质与目标

-   专注于以太坊协议层开发
    
-   涉及共识机制、网络协议与密码学
    
-   不同于上层的智能合约或 DApp 开发
    
-   旨在改进以太坊内部机制与扩容
    

### 参与流程与要求

-   申请流程为自学、填表、面试
    
-   分为有津贴的许可制与无许可制
    
-   面试更看重热情与对以太坊的理解
    
-   最终交付为围绕选定项目的成果
    

### 学习资源与社区

-   核心学习资料为官方规范与维基
    
-   主要讨论在 GitHub 和 Discord 进行
    
-   可借助 AI 辅助理解复杂的英文文档
    
-   所有研究与代码库均为开源
    

## 学习与实践路径

### 基础学习阶段

1.  学习以太坊执行层与共识层
    
2.  理解协议连接与客户端
    
3.  整理学习笔记并定期更新
    

### 方向选择与贡献

1.  选择执行层或共识层小方向
    
2.  在 GitHub 上寻找并修复小 issue
    
3.  提交范围小、易于 Review 的 PR
    

### 项目深耕阶段

1.  选定可深耕的问题域
    
2.  围绕 2-3 个相关 issue 持续贡献
    
3.  最终完成项目提案与交付

# 入门学习路径

-   资料选择：优先从以太坊维基、官方规范文档入手，可借助 AI 工具辅助理解英文文档，也可加入协议学习 Discord 社群求助。
    
-   辅助读物：可参考肖海老师《从 0 到 1 学习以太坊》、小爱老师区块链文明三部曲作为入门辅助读物，降低理解门槛。
    
-   学习特征：EPF 学习曲线相对陡峭，涉及较多数学、密码学内容，无需急于求成，按自身节奏推进即可。
    
-   学习方法：每周整理学习笔记，可将学习进度同步至社媒 @ 官方导师获取反馈，优先熟悉 Python、Go、Rust 等常用语言。
    
-   方向选择：首次参与优先从执行层、共识层方向切入，客户端优先选最成熟的 Go 客户端，再匹配相关 EIP 提案方向。
    

## 贡献实践技巧

1.  PR 提交规范：提交的 PR 需控制修改量，小体量修改更易通过审核，单次提交建议覆盖 2-3 个关联 issue，体现连续性。
    
2.  项目选择逻辑：先选定可长期深耕的问题域，再匹配对应仓库 issue，避免选择浅层次问题导致贡献中途中断。
    
3.  参与时机：当前已进入某期项目 Week3 阶段，零基础参与者无需强行追赶，优先打好基础即可，有基础者可直接参与。
    

### EPF 贡献实践流程

选择问题域（选定可长期深耕的领域）→ 查找（在对应仓库查找关联 issue）→ 开发（控制修改体量，便于审核）→ 提交小体量 PR → 关联（覆盖 2-3 个关联 issue，体现连续性）→ 提交审核 → 审核通过（获得 merge 成为项目成果）

面试为英文形式，无需准备算法、八股等内容，重点展示对以太坊的热情与对特定模块的了解即可。

# 生态与职业发展说明

## 生态发展判断

-   以太坊定位：以太坊当前是全球开发者储备最多的公链，核心竞争力为开发者资源，后续将聚焦 L2 生态与 L1 协议优化。
    
-   生态特征：以太坊生态活跃度高，大量无许可参与者为资深开发者，无需官方资助也可产出高质量贡献。
    

## 职业发展现状

以太坊协议层全职岗位较少，多数贡献者为业余时间参与，以兴趣驱动为主，经济回报稳定性相对较低。

## 常见疑问解答

协议层开发与上层钱包、DApp 无直接关联，零基础入门到能看懂公开研究内容通常需要 2-3 个月。

五、任务4：链上产品基础认知

（一）链上产品 vs 普通互联网产品核心区别

1\. **数据存储底层不同** - 传统互联网：数据、资产记录存企业私有服务器，平台可修改、冻结、删除数据 - 链上产品：资产、交易、合约关键操作上链，分布式账本存储，全网可查，难以篡改

2\. **透明度与可追溯性** 链上转账、合约交互、领测试币等全部生成可查交易记录，包含收发地址、Gas、时间、交易状态；传统平台后台数据不对外公开

3\. **数据区分：并非全部内容都上链** - 上链内容：资产、交易行为、智能合约规则（核心价值数据） - 中心化存储：前端页面、图片、普通用户信息，仍走传统服务器

4\. **资产控制权** 传统平台账号资产归平台管控；链上资产由用户私钥全权掌控，平台无法封禁、没收

（二）智能合约商业模式（合约盈利方式） 智能合约类比自动售货机：预设固定规则，自动执行资金分发、扣费逻辑，盈利途径共6类：

1\. **交易手续费** 开发交易、借贷类工具，用户每笔交互抽取手续费

2\. **发行原生代币** 依靠生态流动性、治理权赋予代币价值，通过代币流通获利

3\. **搭建DeFi协议** 开发借贷、质押、币币兑换等去中心化金融基础设施

4\. **NFT/链游道具业务** 主导NFT铸造、发售、版税分成，游戏链上道具发行售卖

5\. **合约开发外包** 承接项目定制合约（ERC20、质押、空投合约等），按项目收取技术服务费，类似网页开发外包

6\. **合约安全审计服务** 为项目排查合约漏洞，属于行业“卖铲子”刚需服务

核心总结 合约只是技术工具，不会自动赚钱；盈利前提：拥有用户使用合约，或客户付费委托开发/审计；项目长期存活关键：商业模式、流量、用户信任、合规经营。
