## 重要的区块链概念

### 区块链到底是怎么运作的？

每条区块链都是从创世状态开始（Genesis State），也就是网络第一次正式上线的那刻

**节点**

区块链的运行分布于世界各地的不同电脑上，气球中每一台电脑都叫一个节点，每个节点保存**一份完整的交易历史。**

新交易发生的时候，每个节点会独立的验证它。分布式系统的优势就是没有单点故障。

有一种节点叫做验证者，负责提议新的区块。同时，验证者会质押代币作为抵押。

**区块与链**

交易不是一笔一笔单独记录，而是被打包成区块。每个区块包含一批被验证过的交易。

每个区块都通过密码学的方式与前一个区块相连，使得他有不可篡改的特点。

### 共识

世界各地成千上万个节点各自保存着同一份数据的副本。那它们如何在「什么是真的」上达成一致？这就是 **共识（consensus）** 要解决的问题。

### Proof of Work and Proof of Stake

达成共识主要有 Proof of Work 和 Proof of Stake 两种方式。

Proof of Work 是最早期的共识机制，被 Bitcoin 和早期 ETH 使用。该机制要求矿工节点竞争计算一个很难的数学题，优先解出的矿工有权构建下一个区块并获得奖励。由于数学题的复杂性，它使得攻击网络需要耗费大量计算资源，但也要求矿工消耗巨量电能

Proof of Stake 是 ETH 和 Monad 现在都采用的机制。而是通过代币质押参与区块生产。他们会随机抽取去提议新区块或对其他人提议的区块进行投票。抵押使得如果验证者作弊，则会罚没抵押的资产。

### Gas

Gas 是你在区块链上**执行交易**所支付的费用

在 EVM 兼容链上，每一个动作（包括发送代币、部署合约、调用函数）都需要计算。计算可以用 gas units 衡量。

`gas fee = gas units used x gas price`

gas price 会根据网络需求波动。大量用户同时想交易时，gas price 上升；反之则下降。

gas 的作用:

1.  作为奖励支付给处理你交易的验证者
    
2.  防止垃圾信息
    

### 最终性

最终性：指的是在区块链上一笔交易被永久结算、不再可被撤回的那一刻。

### 工具集

**Chain ID**

一个用来标识某条区块链的唯一编号

**RPC（Remote Procedure Call，远程过程调用）：是你的钱包或应用与区块链对话的一个 URL（类似电话号码的作用）**

**Faucet：**免费发放 testnet 代币的网站

**Block Explorer：**查询链上任何东西的网站

# AI Agent 如何拥有支付能力？

## 嘉宾自我介绍

来自 FluxA，负责基础生态。让 Agent 能够非常安全的花钱是非常重要的。而且对于 Agent 来说它需要完成一系列的操作。比如 Agent 会进行一些小微的操作（比如 0.01USD），这对于传统支付来说手续费特别贵，同时 Agent 也有一个问题是它没有身份证（鉴权问题），这个体系是没法办银行卡的。主要分享的内容是为了达到这个目的，需要做哪些**基础设施。（让 Agent 走进商户，个人的实际支付场景）**

## 现状

目前 Agent 进行支付场景存在如下问题：

1.  Identity
    
2.  Authorization
    
3.  Credentials
    
4.  Speed
    
5.  Granularity
    

比如在 chatbox 上输入自己的身份信息是比较危险的

FluxA 不仅包含了钱包，而是一整套协议（最重要的是风控），主要涉及如下的场景

1.  Agent to Agent
    
2.  Agent to Merchant
    
3.  Agent to Tool
    
4.  Transfers and Tipping
    

核心团队是蚂蚁出来的

我觉得比较有新意的是 mandates，有点像给 agent 一部分预算，然后预算内的消费都由 agent 自行处理

其中比较有意思的是 x402 payment。

这是一次性的 agent card

layer 2 和 layer 1 之间的关系是什么（嘉宾说 layer 2 处理的交易会最后集中到 layer 1 进行确认）
