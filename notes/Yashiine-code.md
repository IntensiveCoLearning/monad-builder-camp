**今日学习笔记：Web3 求职、运营与项目实践**

**一、核心结论**

Web3 求职不能只靠海投简历，更重要的是：

`基础能力 + 项目经验 + 圈子曝光 + 持续输出 = 更容易获得机会`

目前 Web3 实习机会没有以前好找，尤其技术岗和运营岗都更依赖内推、社群、线下活动、黑客松和真实项目经历。

**二、Web3 实习怎么找**

比较有效的路径：

-   关注 TG 求职群、Web3 招聘群，先观察岗位 JD，理解岗位真实需求
    
-   多参加线下活动、黑客松、会议，建立真实连接
    
-   优先争取内推，但前提是自己有项目、能力或个人标签
    
-   在社群中持续出现，让别人知道你在认真学习和参与 Web3
    

重点不是“投了多少简历”，而是有没有让别人信任你、记住你、愿意推荐你。

**三、Web3 运营岗需要什么**

运营岗不是单一工作，可以分为：

-   用户增长：拉新、转化、提升用户数量
    
-   社群运营：维护群氛围、组织活动、提升活跃度
    
-   活动运营：策划 AMA、分享会、任务活动、空投活动等
    
-   内容运营：在 Twitter、小红书、抖音、TG 等平台输出项目观点或行业内容
    

新人可以先从“自己起号”开始：

-   找一个清晰定位
    
-   持续发布内容
    
-   分享 Web3 / AI / 项目学习记录
    
-   积累数据后写进简历
    

简历中可以体现：运营账号、内容数据、活动经历、社群贡献、黑客松经历、项目经历、对某个 Web3 方向的理解。

**四、Web3 技术路线**

技术方向可以从三步开始：

1.  学 Solidity 基础语法  
    先理解智能合约怎么写。
    
2.  学 DApp 开发  
    掌握前端如何连接钱包、调用合约、完成链上交互。
    
3.  做项目实操  
    Web3 更看重能不能做出东西，而不是只会背概念。
    

学校课程如果只讲哈希算法、比特币结构，但缺少项目实践，对就业帮助有限。真正重要的是把知识转化成作品。

**五、个人 IP 与内容输出**

个人账号可以成为展示能力的作品集。

可选定位：

-   技术型：合约学习、DApp 项目、开发过程、踩坑记录
    
-   运营型：活动复盘、社群增长、用户增长思考
    
-   Research 型：项目分析、赛道观察、数据分析、趋势判断
    
-   学生成长型：从 0 学 Web3、训练营、项目、找实习过程
    

核心目标是让别人看到：你正在认真进入 Web3，并且有持续产出。

**六、Web3 安全意识**

新人尤其要警惕钓鱼网站和钱包授权风险。

常见风险包括：

-   点陌生链接
    
-   连接不明网站
    
-   授权恶意合约
    
-   钱包资产被转走
    
-   U 卡 / 礼品卡被盗
    
-   交易所、邮箱、身份信息被关联攻击
    

安全原则：

陌生链接不要点，钱包不要乱连，授权一定要看清楚。Web3 资产一旦被链上转走，追回非常困难。

**七、区块链专业与就业现实**

即使是区块链专业，最后真正进入区块链行业的人也不多。很多人会转向普通程序员、AI、销售、主播、考研、考公或其他互联网岗位。

原因是 Web3 岗位本身不多，同时对项目经验、圈子资源和实战能力要求较高。

所以比专业名称更重要的是：

-   项目经验
    
-   实操能力
    
-   社群连接
    
-   持续学习能力
    

**八、案例：喵星智能体**

这是一个 AI + Web3 + 宠物硬件项目。

核心想法：

通过摄像头、声音识别和 AI 分析猫咪行为与叫声，判断猫咪需求，比如饿了、想吃东西或状态异常。

项目流程：

1.  采集猫咪声音 / 行为数据
    
2.  用 AI 分析猫咪意图
    
3.  如果模型不确定，进行多轮确认
    
4.  推送给主人审批
    
5.  通过预言机和智能合约连接支付、审批和履约流程
    

**九、为什么要用区块链**

这个案例提醒我们：Web3 项目不能为了上链而上链，必须说明区块链解决了什么现实问题。

区块链在这个项目中的作用可能包括：

-   支付流程
    
-   链上存证
    
-   订单记录
    
-   权限审批
    
-   透明履约
    
-   多方共享可信数据
    

**十、对学生最有用的行动路径**

更现实的成长路径是：

`学习基础 → 做一个项目 → 参加社群 / 黑客松 → 输出内容 → 认识人 → 获得内推或合作机会`

接下来最值得积累的东西：

-   一个能讲清楚的项目
    
-   一段持续学习记录
    
-   一个内容账号或作品集
    
-   一些社群参与经历
    
-   一点行业方向判断
    
-   基础的 Web3 安全意识

本次分享主要介绍了 AI Agent 支付基础设施的设计思路和落地场景。核心观点是：未来 Agent 会承担大量重复性任务，而这些任务中必然会涉及资金流转，因此“让 Agent 安全、可控地花钱”会成为 Agent 经济的重要基础设施。

分享中提到，Agent 支付的关键不只是完成交易，而是要解决身份识别、预算控制、风控约束、交易可审计和可撤销等问题。比如通过共管式 Agent 钱包、一次性 Agent 卡、与 VISA 合作的长期 Agent 卡，以及基于 X402 协议优化的支付能力，让 Agent 可以在用户授权和预算范围内自动完成支付，同时避免乱花钱或高风险交易。

文档还介绍了几类具体方案：一是开发者可以把 API、MCP、Skill 等能力上架变现；二是针对电商等场景生成支付链接；三是 Agent 之间可以通过唯一 ID 进行点对点转账。当前主要落地在小额支付、USDC/稳定币、Stripe 法币充值、Agent 抢红包、票务预订、行业调研 Skill 等场景。

我的理解是，Agent 支付的难点不在“能不能付款”，而在“如何让付款可信、可控、合规”。这套体系本质上是在为未来 Agent 大规模参与商业活动做准备。随着用户信任、行业标准和合规体系逐渐完善，Agent 支付可能会像早期支付宝一样，从小额场景逐步走向更广泛的日常交易场景。

\# 2026-07-08｜Monad Testnet 智能合约部署学习笔记

\## 今日目标

完成 Week 1 智能合约实践任务：使用 Remix 编写并编译一个最小智能合约，将其部署到 Monad Testnet，并完成一次 read function 和一次 write function 调用。

本次实践重点是理解完整链路：

合约源码 → 编译 → 连接钱包 → 部署到 Monad Testnet → 获得合约地址 → 调用 read/write function → 使用交易 hash 进行验证

\## 使用工具

\- Remix IDE

\- MetaMask 钱包

\- Monad Testnet

\- Monad 测试币 MON

\## 合约说明

本次部署的合约是一个简单的留言板 `MessageBoard`。

合约支持三个主要功能：

\- `postMessage(string content)`：发布一条留言，需要发起链上交易。

\- `getMessageCount()`：读取当前留言数量，不需要 gas。

\- `getMessage(uint256 index)`：读取指定编号的留言内容。

\## 部署过程

1\. 在 Remix 中创建 Solidity 合约文件。

2\. 使用 Solidity 编译器完成编译。

3\. 在 MetaMask 中添加并切换到 Monad Testnet。

4\. 从 Faucet 领取测试 MON。

5\. 在 Remix 的 Deploy 面板中选择 MetaMask 作为连接钱包。

6\. 确认网络为 Monad Testnet 后部署合约。
