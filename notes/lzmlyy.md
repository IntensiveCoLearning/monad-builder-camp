# DAY 2｜工具准备与 Builder 身份：我如何开始远程协作？

## 一、今日学习目标

今天的核心目标不是马上写代码，而是先把进入 Web3 协作环境所需要的基础工具、账号和工作方式准备好。Web3 的学习不是只看教程，更重要的是进入社区、参与讨论、持续输出，让自己的学习过程变成可以被别人看到的 proof of work。

通过今天的学习，我主要完成了三个方向：

1.  了解 Web3 远程协作常用工具；
    
2.  理解 Builder、DevRel、社区参与者等角色的基本定位；
    
3.  开始思考如何把每天的学习记录沉淀成公开或半公开作品。
    

## 二、Web3 常用工具准备

Web3 的工作方式非常依赖远程协作和异步沟通。实习手册里提到，远程协作不能只靠聊天记录，关键结论需要沉淀到文档、Issue 或协作平台中，并标注时区、响应窗口和行动项。

今天我重点整理了以下工具：

## 1\. X / Twitter

X 是 Web3 项目发布信息、生态动态、开发者交流的重要平台。很多项目方、投资机构、开发者和社区成员都会在 X 上发布最新进展，所以它不仅是社交工具，也是了解行业趋势的入口。

我的使用目标：

-   关注 Web3 项目方、开发者、DevRel、研究员；
    
-   记录自己的学习进度；
    
-   后续可以发布简单的学习笔记或项目进展；
    
-   逐步建立自己的 Builder 身份。
    

## 2\. Telegram

Telegram 是 Web3 社区常用的即时通讯工具。很多项目社区、学习群、生态组织都会使用 Telegram 进行通知、交流和活动组织。实习手册也提醒，Telegram 使用时要注意手机号隐私、钓鱼链接、假客服和恶意文件等安全风险。

我的使用原则：

-   不随便点击陌生链接；
    
-   不泄露验证码、密码、助记词；
    
-   尽量使用用户名参与社区；
    
-   先观察社区规则，再发言。
    

## 3\. Discord

Discord 更像是 Web3 项目的社区工作站。一个项目通常会有不同频道，比如公告、技术讨论、中文区、活动区、开发者支持区等。实习手册中也提到，社区管理岗位会经常使用 Telegram、Twitter/X、Discord 等平台维护社区互动和增长。

我的使用目标：

-   加入 Web3 学习社区；
    
-   观察别人如何提问和贡献；
    
-   参与 Co-learning；
    
-   把学习问题和笔记转化成讨论内容。
    

## 4\. GitHub

GitHub 是开发者参与开源协作的重要平台。它可以进行代码托管、版本控制、Pull Request、Issue 管理和项目协作。实习手册中提到，GitHub 支持版本控制、协作开发、代码审查、项目管理和开源贡献。

我的准备方向：

-   完善 GitHub 个人主页；
    
-   创建 Web3 学习记录仓库；
    
-   以后把笔记、代码、项目 Demo、学习路线放进去；
    
-   让自己的学习过程留下可追踪记录。
    

## 5\. Notion / Google Docs

Notion 和 Google Docs 更适合做长期笔记、协作文档和学习记录。Web3 远程协作强调信息留痕，所以学习过程不能只存在聊天里，而应该沉淀为文档。

我的记录方式：

-   每天写 Day Build Log；
    
-   记录学习目标、学习内容、问题、截图、链接；
    
-   重要概念单独整理；
    
-   每周做一次复盘。
    

## 6\. AI Coding 工具

我可以根据自己的习惯选择 Cursor、Claude Code、Codex、ChatGPT 等 AI Coding 工具。它们可以帮助我理解代码、生成项目骨架、调试错误、总结文档和加快学习速度。

但今天我也意识到，AI 工具不能代替自己理解。比较好的方式是：

-   先自己明确需求；
    
-   再让 AI 帮我拆解步骤；
    
-   代码生成后必须自己看懂；
    
-   把 AI 辅助过程也记录进 Build Log。
    

## 三、我对 Builder 身份的理解

今天最重要的认知是：Web3 里的 Builder 不只是“会写代码的人”，而是持续参与建设、输出、协作和连接生态的人。

从岗位全景图来看，Web3 不只有智能合约工程师，也有前端、后端、产品运营、社区管理、研究分析等多种岗位。比如前端工程师需要支持钱包连接、交易签名、链上数据展示和智能合约交互；社区管理则需要维护 Telegram、X、Discord 等平台的用户互动。

所以我现在可以先从低门槛的 Builder 路线开始：

-   学习基础概念；
    
-   参与社区讨论；
    
-   写学习笔记；
    
-   记录自己的问题和解决过程；
    
-   尝试做小项目；
    
-   慢慢积累公开作品。
    

我现在不需要一开始就假装自己很懂 Web3，而是要持续留下学习证据。Proof of work 的价值在于：别人可以看到我学了什么、做了什么、解决了什么问题，以及我是否能持续进步。

-   **Web3 不是单纯炒币**  
    Web3 更像是一套新的互联网协作方式：资产、身份、数据、组织都可以放到链上运行。新人不要一上来只看币价，先理解区块链、钱包、智能合约、Dapp 和安全风险。
    
-   **区块链是什么**  
    区块链可以理解为公开、连续、难篡改的账本。每个区块记录交易，并通过哈希连接前一区块，所以一旦修改历史记录，后面的区块都会受影响。
    
-   **为什么说区块链不可篡改**  
    区块链不是靠某一个公司保存数据，而是由很多节点共同保存账本。只改一台机器没用，因为其他节点会发现数据不一致，这就是分布式账本的核心价值。
    
-   **公开透明不等于完全暴露隐私**  
    链上交易是公开的，别人能看到某个钱包地址的余额和交易历史。但如果地址没有和真实身份绑定，别人通常不知道这个钱包背后是谁。
    
-   **Web3 和 Web2 最大区别**  
    Web2 里账号和数据主要掌握在平台手里，平台可以封号、改规则。Web3 更强调用户自己掌握资产和身份，用钱包、私钥和智能合约完成交互。
    
-   **Web3 不等于 Web 3.0**  
    Web3 通常指区块链驱动的去中心化互联网；Web 3.0 更偏语义网、结构化数据和机器可读。两个词很像，但技术路线和行业语境不一样。
    
-   **公链、联盟链、私链区别**  
    公链任何人都能参与，透明度高但效率低；联盟链由多个机构共同维护，适合金融、供应链；私链由单一组织控制，效率高但去中心化弱。
