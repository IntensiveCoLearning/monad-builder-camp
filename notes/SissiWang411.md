**一、支付能力的风险控制**

l

l **金融场景的特殊要求**：在金融场景中， AI产生幻觉导致乱花钱的行为绝对不能接受，必须建立严格的风控机制。

l **审计必要性**：每一笔交易都需要可被审计且流程清晰，这是金融逻辑中最核心的部分。

l **撤销功能**：当用户对交易不满时，需要提供完整的交易撤销解决方案，这是金融链路中不可或缺的环节。

![](blob:https://intensivecolearn.ing/292a6d84-3cec-4243-9e18-c3e1abd89d5d)

1. 支付解决方案组件

l

l **Agent   Card**：与VISA深度合作推出的专为Agent设计的信用卡，无需身份证即可使用，实现真实消费场景。

l **Agent Charge**：针对不同商家或个体设计的收付款解决方案，支持多样化支付场景。

l **Monetize功能**：开发者可将开发的skill或MCP在平台上架，实现知识变现，每次调用产生收益归开发者所有。

2\. 嵌入式授权协议

l **核心机制**：通过embedded  mandate with different settlement协议，用户签署授权后， Agent可在预算范围内自由交易，无需逐笔人工确认。

l **优势**：避免每笔交易都需要人工审批导致的流程冗长问题，更符合Agent自主运行的业务逻辑。

l **典型场景**：例如设定50元预算后， Agent可在此范围内自主完成多笔小额交易。

  

![](blob:https://intensivecolearn.ing/b8d5b9a4-8236-4f64-a8b3-8d254964aa7f)

**二、共管钱包**

l

l **共管模式**：采用人机共管模式，既不是完全自主也不是完全托管，平衡了灵活性与安全性。

l **即时撤销**：支持一键撤销交易，无需修改底层代码，操作便捷性远超传统AI支付方案。

l **离线授权**：预先设置授权策略， Agent可在无人值守时继续工作，真正实现72小时不间断服务。

1\. 钱包核心功能

l **预算控制**：人类设定规则， Agent在规则范围内自主运作，包括单笔限额、总额控制等。

l **审计追踪**：自动记录完整交易日志，支持手动和自动两种审计追踪模式。

l **策略预置**：支持预先授权策略设置，确保Agent行为符合预期，同时保持操作灵活性。

2. 设计理念

l **平衡原则**：在完全自主与完全托管之间找到最佳平衡点，既保证安全性又不牺牲效率。

l **自然交互**：操作界面设计符合人类直觉，撤销等关键功能实现"一键式"操作。

l **持续服务**：通过合理的授权机制设计，确保Agent能够真正实现7×24小时不间断服务。

**三、**[**HTTP**](HTTP)**原生支付协议**

l

l **协议基础**:  [HTTP](HTTP)  402状态码是互联网原生的支付协议，为Agent提供直接调用支付功能的底层支持。

l **自动化特性** : 通过该协议， Agent只需在初始阶段获得人类批准（approve），后续即可自主完成支付流程，实现完全自动化。

l **协议优化**:  FluxA在去年推出的针对Agent特定场景的协议基础上进行了优化提升，使其更适合AI代理的支付需求。

**四、风险控制引擎**

l

l **意图支付机制**:

o  **工作流程** : Agent提出支付意图→ 用户批准意图 → 风险引擎控制执行

o  **优势**:     只需一次批准即可完成整个任务链的支付，避免传统支付中每笔交易都需要人工干预的繁琐流程

l **风控设计** :

o  **严格性要求**:    涉及金融交易时对风控敏感度和准确性要求极高，特别是在生产环境中

o  **策略工程**: 采用policy  engineering构建的风控引擎，通过预设约束条件确保交易安全

o  **执行控制**:    在预算范围内的合规支付自动执行，超出预算或不符合任务目标的支付会被钱包层直接拦截

**十五、问答环节**

1. 支付链路与钱包形态

l **双链路支持**：同时支持法币（通过Stripe）和稳定币（区块链钱包）两种支付方式，法币充值会转换为平台积分系统

l **钱包功能**：通过安装skill并授权Google账号登录后，可获得区块链钱包功能，支持充值和收款操作

l **形态特点**：当前以区块链钱包形态为主，地址间可直接进行资金转移

2. 与其他公司合作情况

l **竞品关系**：与Coinbase 、Ripple是生态伙伴而非直接竞争，作为Coinbase   facility上最大交易量用户

l **技术基础**：基于Coinbase推出的X402协议进行二次开发和协议创新

l **合作深度**：与VISA等传统支付公司建立深度合作关系，共同探索支付场景创新

3\. agent card模式与传统银行卡

l **创新特点**：针对支付场景开发一次性卡模式，相比传统借记卡体系有差异化设计

l **成本挑战**：大规模使用时一次性卡成本较高，需在底层架构进行优化

l **合作必要性**：与VISA合作的关键原因在于降低长期运营成本，实现规模化应用

4\. agent开发所需技能

l **核心要求**：需要对整个开发流程有全面理解，而非特定技术栈

l **知识广度**：前端开发者需补充后端数据结构知识，后端需理解前端交互逻辑

l **适应趋势**：在AI辅助编程时代，更强调对计算机系统结构的整体把握能力

5\. AI agent VISA card的KYC问题

l **豁免场景**：当B端已完成KYC时， Agent Card可免去重复验证

l **底层逻辑**：因Agent本身无独立身份(identity)，采用ERC8004等协议处理特定支付场景

l **实施难点**：现有协议在具体实现场景中仍存在操作性问题待解决

6\. agent付款的收款结构

l **简化流程**：安装对应skill后支持直接转账或通过payment link完成支付

l **体验优化**：提供红包等社交化支付功能，降低使用门槛

l **技术实现**：通过特定参数(p)传递支付能力，实现快速部署

7\. AI的付费能力

l **转账方式**：支持点对点直接转账基础功能

l **高级功能**：通过创建mandate授权支付指令，实现自动化付款

l **双向操作**： payment link兼具收款和付款双重功能

8. 支付链的选择

l **当前选择**：主要基于Base链使用USDC稳定币，因其交易量最大

l **跨链支持**：平台已具备跨链操作能力，但默认不开启

l **协议中立**：技术上可支持多链，实际选择取决于用户需求和协议发展

9\. 小微支付与大额支付

l **当前重点**：主要服务小微支付场景（<$10万）

l **信任构建**：大额支付依赖用户对Agent支付体系的长期信任积累

  

l **发展类比**：类似支付宝早期发展路径，需经历信任建立过程

10. 防范洗钱与支付牌照

l **风控措施**：规划引入KYC/KYB等身份认证机制防范洗钱

l **牌照策略**：通过与VISA等持牌机构合作解决监管合规问题

l **发展阶段**：当前需求规模较小，但合规建设已纳入业务规划

11\. agent自主付费与产品推广

l **推广痛点**：解决Agent能力被发现效率低下的问题

l **平台方案**：与百度合作建立集中式listing平台，优化发现流程

l **职业路径**： FinTech学生可通过Agent开发进入支付领域，需补充工程实现能力

12\. agent支付的主流趋势

l **判断依据**：基于AI技术在过去两年带来的深度生活变革，特别是OpenAI产品的突破性进展

l **不可逆性** ：Agent                commerce被认为是必然发展趋势，需要创业者保持坚定信念（bias）和支撑依据（back）

l **类比说明**：类似自动驾驶技术，虽不会完全取代人类，但会接管大部分重复性操作环节

1）基础设施对比

l **发展现状**： Base链发展速度远超以太坊主网，成为当前最完善的区块链支付基础设施

l **核心优势**：

o  决策和执行效率高于受基金会流程限制的以太坊

l **合规特性**： USDC在Base链上的合规优势（美国监管认可）

2）支付场景演化

l **生活化应用**：

o  水电煤等日常费用缴纳

o  通过Agent实现自动化账单支付

l **技术特性**：

o  支付链路透明度高（不同于AI黑箱）

o  遵循XCni协议规范

o  与传统支付系统具有可比性
